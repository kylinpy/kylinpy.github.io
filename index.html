第一次用hexo與GitHub Pages

因為是第一次使用，在這裡做個紀錄當作開始

自己是個剛學python沒多久的程式新手，也都是跟著網路大神走

這邊是下面參考影片的大神部落格:Re:code

NPM(Node Package Manager) 是node.js的套件管理器，所以須先安裝Nodejs

傳送門: nodejs下載網址

(1)在這裡我選擇了v20.13.1 (LTS)
(2)安裝完之後,在終端機中輸入npm檢查有沒有安裝成功(可以用cmd或是powershell)

傳送門: Hexo官網

(1)新建資料夾,在資料夾內以vscode開啟
關於這個部份我因為安裝vscode的時候沒有設定到這個功能,所以是用shift+右鍵開啟PowerShell之後輸入code開啟vscode
(2)按ctrl + ~鍵，vscode會在該目錄下開啟終端機
(3)檢查路徑是否為剛剛所建立的資料夾,若否則利用cd 路徑 進行移動(路徑用資料夾路徑複製貼上即可)
(4)將Hexo官網要我們輸入的5行指令

在這裡我發生了一個錯誤:

“因為這個系統上已停用指令碼執行，所以無法載入 C:\Users\USER\AppData\Roaming\npm\hexo.ps1 檔案。”
根據下方參考文章表示，這是因為Windows的執行政策上設定預設的安全限制

解決辦法:

在PowerShell點右鍵選擇以系統管理員身分執行
輸入Set-ExecutionPolicy RemoteSigned
按下[A]
然後關閉PowerShell
(5)回到vscode的終端機繼續執行剛剛的指令就可以了

(1)執行完官網的5個指令之後，左方會出現設定的資料夾名稱下方有個blog(若沒看左列按一下Explorer)
(2)下方出現 INFO Hexo is running at http://localhost:4000/ . Press Ctrl+C to stop.
(3)連結至該網址就可以看到建立的blog

(1)於左列blog下的source_posts是建立文章的地方
(2)點選一開始的hello-world.md進行編輯
(3)裡面的語法是 markdown
傳送門: MarkDown語法大全
傳送門: Markdown官方教程(簡體中文版)

(4)查看hexo提供的功能: hexo help
(5)建立新文章(新檔案): hexo new 檔名
(6)開啟建立的blog:hexo serve 連結至伺服器
(7)

參考影片: 十分鐘小魔法-Hexo + GitHub
參考文章: 是Ray不是Array
